## 版本号规则

采用[语义化版本](https://semver.org/lang/zh-CN/)（Semantic Versioning）：

```
主版本号.次版本号.修订号
```

- **主版本号（MAJOR）**：不兼容的 API 变更、内容结构重大调整
- **次版本号（MINOR）**：新增主题章节、新增代码模块，向下兼容
- **修订号（PATCH）**：文档勘误、代码 Bug 修复、内容微调

## 发布流程

1. 确认变更内容，更新本文件中的版本历史
2. 更新版本号至新版本
3. 提交 Git Tag：`git tag v<主>.<次>.<修订>`
4. 推送 Tag：`git push origin v<主>.<次>.<修订>`
5. 基于 Tag 创建 GitHub Release，附上变更说明
